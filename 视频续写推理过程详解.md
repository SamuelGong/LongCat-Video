# 视频续写（Video Continuation）推理过程详解

本文档详细分析 `run_demo_video_continuation.py` 的完整推理流程，重点说明视频续写与文本生成的区别。

---

## 一、整体流程概览

视频续写脚本生成**三次视频**，每次都是基于输入视频的条件生成：

1. **第一次生成：基础VC** (`output_vc.mp4`) - 480p，50步
2. **第二次生成：蒸馏VC** (`output_vc_distill.mp4`) - 480p，16步
3. **第三次生成：精炼VC** (`output_vc_refine.mp4`) - 720p，50步

**核心区别**：使用 `generate_vc()` 方法，需要输入视频作为条件帧。

---

```python
latents = self.prepare_latents(
```

**`prepare_latents` 内部处理** (`pipeline_longcat_video.py:215-283`):

```python
# 1. 计算潜在空间的帧数
num_latent_frames = (num_frames - 1) // self.vae_scale_factor_temporal + 1
# (93-1) // 4 + 1 = 24帧（潜在空间）

# 2. 初始化随机噪声（用于新生成的帧）
shape = (batch_size, num_channels_latents, num_latent_frames, ...)
latents = torch.randn(shape, ...)  # [1, 16, 24, 60, 104]

# 3. 编码输入视频的条件帧
condition_data = video[:, :, -num_cond_frames:]  # 取最后13帧
# Shape: [1, 3, 13, 480, 832]

# 4. VAE编码条件帧到潜在空间
cond_latents = []
for i in range(batch_size):
    encoded_input = condition_data[i][:, -num_cond_frames:].unsqueeze(0)
    latent = retrieve_latents(self.vae.encode(encoded_input), gen)
    cond_latents.append(latent)

cond_latents = torch.cat(cond_latents, dim=0).to(dtype)
cond_latents = self.normalize_latents(cond_latents)
# Shape: [1, 16, num_cond_latents, 60, 104]
# num_cond_latents = 1 + (13-1) // 4 = 4帧（潜在空间）

# 5. 将条件帧的latents填充到初始噪声的前面
num_cond_latents = 1 + (num_cond_frames - 1) // self.vae_scale_factor_temporal
latents[:, :, :num_cond_latents] = cond_latents
# 前4帧是条件帧（编码自输入视频），后20帧是随机噪声（待生成）
```

**关键点**:
- **条件帧编码**: 输入视频的最后13帧被VAE编码为4个潜在帧
- **潜在空间布局**: `[条件帧(4帧) | 随机噪声(20帧)]` = 24帧
- **像素空间对应**: 4个潜在帧对应13个像素帧（VAE时间下采样4倍）

##### Step 3: KV Cache初始化（优化）

```python
if use_kv_cache:
    cond_latents = latents[:, :, :num_cond_latents]  # 提取条件帧
    # 预计算条件帧的KV，避免在去噪循环中重复计算
    self._cache_clean_latents(cond_latents, max_sequence_length, ...)
    kv_cache_dict = self._get_kv_cache_dict()
    latents = latents[:, :, num_cond_latents:]  # 只处理新帧部分
```

**KV Cache原理**:
- 条件帧的Key-Value在第一次计算后缓存
- 后续50步去噪中，条件帧的KV可以复用
- **目的**: 减少重复计算，加速推理

##### Step 4: 去噪循环（50步）

**位置**: `pipeline_longcat_video.py:1030-1079`

```python
for i, t in enumerate(timesteps):  # 50步迭代
    # 准备模型输入
    latent_model_input = torch.cat([latents] * 2) if self.do_classifier_free_guidance else latents
    # 如果使用KV cache，latents只包含新帧部分 [1, 16, 20, 60, 104]
    # 如果不使用KV cache，latents包含所有帧 [1, 16, 24, 60, 104]
    
    # 时间步处理
    timestep = t.expand(latent_model_input.shape[0]).to(dit_dtype)
    timestep = timestep.unsqueeze(-1).repeat(1, latent_model_input.shape[2])
    if not use_kv_cache:
        timestep[:, :num_cond_latents] = 0  # 条件帧的时间步设为0（不参与去噪）
    
    # DIT模型前向传播
    noise_pred = self.dit(
        hidden_states=latent_model_input,
        timestep=timestep,
        encoder_hidden_states=prompt_embeds,
        encoder_attention_mask=prompt_attention_mask,
        num_cond_latents=num_cond_latents,  # ← 关键：告诉模型哪些是条件帧
        kv_cache_dict=kv_cache_dict  # ← 关键：传入KV缓存
    )
    
    # CFG引导（与T2V相同）
    if self.do_classifier_free_guidance:
        noise_pred_uncond, noise_pred_cond = noise_pred.chunk(2)
        noise_pred = noise_pred_uncond * st_star + guidance_scale * (noise_pred_cond - noise_pred_uncond * st_star)
    
    # Scheduler更新
    noise_pred = -noise_pred
    if use_kv_cache:
        latents = self.scheduler.step(noise_pred, t, latents, return_dict=False)[0]
        # 只更新新帧部分
    else:
        latents[:, :, num_cond_latents:] = self.scheduler.step(
            noise_pred[:, :, num_cond_latents:], t, 
            latents[:, :, num_cond_latents:], return_dict=False
        )[0]
        # 只更新新帧部分，条件帧保持不变
```

**关键区别（与T2V对比）**:

| 特性 | T2V (`generate_t2v`) | VC (`generate_vc`) |
|------|---------------------|-------------------|
| **输入** | 随机噪声 | 条件帧 + 随机噪声 |
| **条件帧** | 无 | 输入视频的最后13帧 |
| **潜在空间布局** | `[随机噪声(24帧)]` | `[条件帧(4帧) \| 随机噪声(20帧)]` |
| **时间步处理** | 所有帧相同时间步 | 条件帧时间步=0（不参与去噪） |
| **KV Cache** | 不使用 | 使用（优化条件帧计算） |
| **`num_cond_latents`** | 0 | 4（潜在空间） |

##### Step 5: DIT模型中的条件帧处理

**位置**: `longcat_video_dit.py:280-370`, `attention.py:107-147`

在DIT的Attention机制中：

```python
# attention.py:124-137
if num_cond_latents is not None and num_cond_latents > 0:
    # 视频续写模式
    num_cond_latents_thw = num_cond_latents * (N // shape[0])
    # 分离条件帧和新帧的tokens
    q_cond = q[:, :, :num_cond_latents_thw]  # 条件帧的query
    k_cond = k[:, :, :num_cond_latents_thw]  # 条件帧的key
    v_cond = v[:, :, :num_cond_latents_thw]  # 条件帧的value
    
    q_noise = q[:, :, num_cond_latents_thw:]  # 新帧的query
    
    # 条件帧的self-attention（只在条件帧内部）
    x_cond = self._process_attn(q_cond, k_cond, v_cond, shape)
    
    # 新帧的attention（可以attend到所有帧：条件帧+新帧）
    x_noise = self._process_attn(q_noise, k, v, shape)
    # ↑ 注意：k和v包含所有帧，所以新帧可以看到条件帧
    
    # 合并输出
    x = torch.cat([x_cond, x_noise], dim=2)
```

**Attention机制**:
- **条件帧**: 只在条件帧内部做self-attention（保持原样）
- **新帧**: 可以attend到所有帧（条件帧+新帧），实现条件引导
- **结果**: 新帧的生成受到条件帧的引导，保证连续性

##### Step 6: 合并条件帧和新帧

```python
if use_kv_cache:

    latents = torch.cat([cond_latents, latents], dim=2)

```

##### Step 7: VAE解码


**输出**: 93帧视频，其中前13帧来自输入视频（条件帧），后80帧是新生成的。

## 三、关键机制详解

### 3.1 条件帧机制

**目的**: 保证生成的视频与输入视频的连续性。

**实现**:
1. **编码**: 输入视频的最后13帧 → VAE编码 → 4个潜在帧
2. **布局**: `[条件帧(4帧) | 随机噪声(20帧)]`
3. **去噪**: 条件帧时间步=0（不参与去噪），新帧正常去噪
4. **Attention**: 新帧可以attend到条件帧，实现条件引导
5. **解码**: 所有帧一起解码，条件帧保持原样

### 3.2 KV Cache优化

**原理**:
- 条件帧的KV在第一次计算后缓存
- 后续50步去噪中，条件帧的KV复用
- 只对新帧部分重新计算KV

**效果**:
- 减少计算量：条件帧的KV只计算一次
- 加速推理：特别是在长视频生成中效果明显

### 3.3 时间步处理

```python
timestep = t.expand(latent_model_input.shape[0]).to(dit_dtype)
timestep = timestep.unsqueeze(-1).repeat(1, latent_model_input.shape[2])
if not use_kv_cache:
    timestep[:, :num_cond_latents] = 0  # 条件帧时间步=0
```

**含义**:
- **条件帧**: 时间步=0，表示"完全干净"，不参与去噪
- **新帧**: 时间步从高到低（1.0 → 0.001），正常去噪

### 3.4 Attention机制

**条件帧的Attention**:
- **Self-Attention**: 只在条件帧内部
- **Cross-Attention**: 可以attend到文本条件

**新帧的Attention**:
- **Self-Attention**: 可以attend到所有帧（条件帧+新帧）
- **Cross-Attention**: 可以attend到文本条件

**结果**: 新帧的生成受到条件帧的引导，保证时序连续性。

## 四、与T2V的对比

| 特性 | T2V (`generate_t2v`) | VC (`generate_vc`) |
|------|---------------------|-------------------|
| **输入** | 文本prompt | 文本prompt + 输入视频 |
| **条件帧** | ❌ 无 | ✅ 输入视频的最后13帧 |
| **潜在空间初始化** | 全随机噪声 | 条件帧编码 + 随机噪声 |
| **时间步处理** | 所有帧相同 | 条件帧=0，新帧正常 |
| **KV Cache** | ❌ 不使用 | ✅ 使用（优化） |
| **Attention** | 所有帧相互attend | 条件帧固定，新帧可attend条件帧 |
| **输出** | 全新视频 | 输入视频 + 续写部分 |
| **用途** | 从零生成 | 视频续写、编辑 |

## 五、数据流转换

```
输入视频 [T_video帧]
  ↓ 取最后13帧
条件帧 [13帧, 480×832]
  ↓ VAE编码
条件潜在帧 [4帧, 60×104]
  ↓ 拼接随机噪声
完整潜在空间 [4条件帧 + 20新帧 = 24帧, 60×104]
  ↓ 50步去噪（条件帧时间步=0）
去噪后的潜在空间 [24帧, 60×104]
  ↓ VAE解码
输出视频 [93帧, 480×832]
  ↓ 后处理
最终视频 = 输入视频 + 新生成的80帧
```

## 七、关键技术点

1. **条件帧机制**: 通过VAE编码输入视频的最后几帧，实现条件生成
2. **KV Cache**: 缓存条件帧的KV，加速推理
3. **时间步处理**: 条件帧时间步=0，不参与去噪
4. **Attention引导**: 新帧可以attend到条件帧，保证连续性
5. **三阶段生成**: 基础 → 蒸馏 → 精炼，平衡速度和质量

---

## 八、总结

`run_demo_video_continuation.py` 实现了**视频续写**功能，核心是通过条件帧机制保证生成的视频与输入视频的连续性。与纯文本生成（T2V）的主要区别是：

1. **条件帧编码**: 输入视频的最后13帧被编码为潜在空间的条件帧
2. **条件引导**: 新帧的生成受到条件帧的引导
3. **KV Cache优化**: 加速条件帧的计算
4. **三阶段生成**: 从基础到精炼，逐步提升质量

这使得模型能够生成与输入视频连贯的续写内容，实现自然的视频扩展。
